Contratos Tipados

AgentIntentContract — o type system do domínio

type AgentIntentContract = { intentId: string; // "approve-designation" actionType: AgentActionType; // APPROVE_PROPOSAL discoveryGate: DiscoveryGate; // quem vê a ação visibility: VisibilityScope; // agent | scoped-data | sensitive phase: TransactionPhase; // single-step | two-phase | impact-preview risk: 'low' | 'medium' | 'high'; prepare: string[]; // dados necessários preview: string[]; // o que o usuário vê antes commit: string; // serviço que executa recovery: string; // como desfazer };

Modelo de Segurança

4 filtros por micro-UI

1. Discovery Filter
Se o perfil não pode executar → UI não aparece (invisível, não disabled).
canAgentAction(type)
2. Data Visibility Filter
Respeita dataAccessLevel e canSeeSensitiveData. Dados sensíveis só para quem pode.
3. Execution Filter
Abrir micro-UI ≠ autorizar execução. Re-valida permissão no commit.
4. Channel Filter
Comunicação (WhatsApp, S-89) só se canSendZap está ativo no perfil.
3 camadas de enforcement

Supabase RLS (banco) → permissionService.ts (serviço) → usePermissions.ts (hook React). Cache com TTL de 5 minutos. 7 seed policies. Fallback seguro: só tab agente + 6 ações read-only.

A habilidade mais valiosa

Uma Analogia

O restaurante do futuro

👨‍🍳
Antigamente: você era o chef
Entrar na cozinha
Escolher ingredientes
Cortar, temperar, cozinhar
Controlar o fogo
Lavar a louça

= Escrever cada linha de código manualmente
🍽️
Agora: você é o cliente
"Quero algo leve, sem glúten,
com bastante proteína,
para duas pessoas"

O garçom entende. O chef executa.
O resultado chega à mesa.

= Expressar intenção com clareza
A mudança fundamental

Você não precisa saber cozinhar para comer bem. Mas precisa saber o que quer e comunicar suas restrições. "Sem glúten" é uma restrição. "Para duas pessoas" é contexto. "Com proteína" é um requisito.

Um Novo Cargo

O Engenheiro de Intenção

"Agentes de IA não precisam de instruções mais precisas. Precisam de mais contexto e mais liberdade." — Manuel Klein, SQUER, 2026
O que faz
Entrevista stakeholders
Acompanha processos reais (process shadowing)
Mapeia domínios (event storming)
Analisa dados (taxas de erro, tempos)
Transforma tudo em documentos de intenção

Focused Impact Pair:

🎯 Engenheiro de Intenção (o que + por que)
⚙️ Engenheiro de Sistemas (como + orquestra IA)

2 pessoas + IA = equipe completa
O Aviso

O perigo do "Vibe Coding"

⚠️
Vibe Coding
"Pede pra IA e vai"
Sem entender o resultado
Sem testar
Sem verificar segurança

Resultado: funciona até o dia que quebra. E aí ninguém sabe consertar.
✅
IDD (Intent-Driven)
Expressa intenção com clareza
Define restrições e critérios
Valida o resultado
Refina quando necessário

Resultado: compreensão no nível certo. Não da sintaxe, mas do problema.
A diferença em uma frase

Vibe Coding: "Confia na IA e vai." — IDD: "Entende o problema, expressa com clareza, valida o resultado."
